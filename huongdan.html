3 Extraction Pipeline

  1. Nhận file — Lưu tạm, kiểm tra extension + size limit
  2. Extract text — PDF dùng pdfplumber, image dùng Tesseract, TXT đọc trực tiếp
  3. Build prompt — Tạo prompt yêu cầu extract các field theo document_type
  4. Call provider — Mock dùng regex, DeepSeek/Claude dùng API
  5. Parse JSON — Validate output bằng Pydantic schema
  6. Business rules — Kiểm tra total = subtotal + tax, line item math
  7. Confidence scoring — Tính điểm theo field completeness + validation
  8. Human review flag — Nếu confidence < 0.75 hoặc có validation errors

5 Confidence Scoring

Điều kiệnPenalty
Thiếu required field (invoice_number, date, seller, buyer, total, currency)-0.10 / field
Không có line_items-0.08
Mỗi validation error-0.05
Human review: Nếu confidence < 0.75 hoặc có bất kỳ validation error → requires_human_review = true. Production system sẽ đưa vào queue để nhân viên kiểm tra lại.

6 Business Rules Validation

4 business rules được kiểm tra tự động:

  1. Total check — total_amount ≈ subtotal + tax (sai lệch < 2%)
  2. Line item math — amount ≈ quantity × unit_price
  3. Date ordering — invoice_date không được sau due_date
  4. Positive total — total_amount phải dương
